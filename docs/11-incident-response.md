# 11 - Incident Response

## Goals

- Detect security incidents quickly
- Contain the threat
- Eradicate the attacker
- Recover systems safely
- Capture lessons learned

## Incident response steps

1. Detect: monitor logs, alerts, and intrusion tools.
2. Contain: isolate affected systems and stop malicious activity.
3. Eradicate: remove backdoors, malicious files, and compromised accounts.
4. Recover: restore from trusted backups, validate services, and harden the system.
5. Learn: conduct a post-mortem and update the guide.

## Notes

- Preserve forensic evidence when needed.
- Keep incident response playbooks current.
- Use centralized logging and alerting to reduce detection time.
